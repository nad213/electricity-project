Endpoints

Méthode Endpoint Description
GET /api/v1/meta Plages de dates disponibles et listes de référence (filières, pays).
GET /api/v1/courbe_conso Courbe de consommation (puissance, MW). Paramètres : debut, fin.
GET /api/v1/energie_conso Énergie consommée par mois (GWh). Paramètres : debut, fin.
GET /api/v1/courbe_prod Courbe de production par filière (MW). Paramètres : debut, fin, filiere.
GET /api/v1/energie_prod Énergie produite par mois (GWh). Paramètres : debut, fin, filiere.
GET /api/v1/echange Courbe d'échanges (MW). Paramètres : debut, fin, pays (dont total).
GET /api/v1/energie_echange Énergie import/export par mois (GWh). Paramètres : debut, fin, pays.
GET /api/v1/parc Parc installé mensuel éolien/solaire (MW).

Paramètres communs

  • debut / fin — format AAAA-MM-JJ. Plage maximale : 366 jours pour les courbes, 10 ans pour l'énergie.
  • filiere — clé de filière (ex. nucleaire, eolien, solaire…). Voir /meta.
  • pays — clé de frontière (ex. ech_comm_allemagne_belgique…) ou total. Voir /meta.
  • Les endpoints energie_* renvoient un total par mois en GWh.

Exemples

Chaque requête doit porter l'en-tête Authorization: Bearer <clé>. Pratique : exportez votre clé une fois, puis réutilisez-la. 

export ELF_KEY="votre_clé"

Métadonnées :

curl -H "Authorization: Bearer $ELF_KEY" \
  "{{ api_base }}/meta"

Courbe de consommation sur une période :

curl -H "Authorization: Bearer $ELF_KEY" \
  "{{ api_base }}/courbe_conso?debut=2024-01-01&fin=2024-01-07"

Énergie consommée par mois (GWh) sur une année :

curl -H "Authorization: Bearer $ELF_KEY" \
  "{{ api_base }}/energie_conso?debut=2024-01-01&fin=2024-12-31"

Courbe de production solaire :

curl -H "Authorization: Bearer $ELF_KEY" \
  "{{ api_base }}/courbe_prod?debut=2024-06-01&fin=2024-06-07&filiere=solaire"

Énergie échangée import/export par mois (total France) :

curl -H "Authorization: Bearer $ELF_KEY" \
  "{{ api_base }}/energie_echange?debut=2024-01-01&fin=2024-12-31&pays=total"
